HTML

  • HTML is used to create the stucture of the webpage. It is made up of elements which work togetherr to provide content and the basic framework for all pages on the web.
  • Elements are used to divide the content on the page and to describe the content to browsers. Some common element types that are used to divide and describe a page include; Meta-related elements, Content division elements and Semantic elements.
  • The head element contains information about the webpage
  • Inside the head element we can use the following meta data tags:
  • title: this element defines the title of a webpage
  • style: this element defines the CSS styles associated with a website
  • base: element defines the base URL for a webpage
  • link: element contains an external resource to the HTML document
  • The body element represents the visible content shown to the user
  • The body element is an example of a semantic HTML element
  • Some types of element tags include:
  • H1, H2, H3, H4, H5, H6: these represent the level of heading a given text block represents
  • P element: represents a paragraph or block of text
  • ul, ol and li: elements that represent unordered lists, ordered lists and list items.
  • img: element contains information about images that are displayed on the webpage
  • br: element creates a line of empty space, or line break between two blocks of content
  • a element: which stands for anchor, creates links to the same webpage or other webpages

CSS

  • There are 3 ways to style a webpage using CSS: Inline CSS, Internal CSS Style Sheet and External CSS style sheet
  • Use inline CSS when you want to apply a change directly in the HTML file
  • The Internal Style SHeet contains CSS rules for the webpage in the head section of the HTML file.
  • When using an External CSS Style Sheet, we link an external file which has all our CSS styles using the link element placed in the HTMLs head. In this case you will need to make sure to link the CSS file to the HTML file otherwise changes won't be applied.
  • Errors are referred to as bugs and finding those errors and fixing them is called debugging
  • There are 5 main steps to use when debugging:
    1. The developer should compare what the outcome is doing in comparison to what it should be doing
    2. The developer should hunt for the bug in the code to find the problem/what is causing the problem
    3. After the error/bug has been found, the developer will have to find a solution to fix the bug
    4. When the developer has implemented a fix, they will need to test the app to make sure it is working properly and they haven't created new bugs somewhere else in the project
    5. Now that the bug is completely fixed, it's good practise to document the process you used to find the solution. This documentation can be used in future if same issue appears again
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A box-shadow CSS property adds shadow effects around an element's frame
  • Assigning colours in CSS can be done in many ways, including semantic references such as blur or grey. We can also refer to code by its hexadecimal code
